# Graphs
A graph is a data structure, consisting of nodes and links between some of them. There are different types of graphs, such as directed/undirected, weighted/unweighted and multigraphs.
This project is an implementation of an unweighted undirected graph, a weighted undirected graph, an unwaighted directed graph and a weighted directed graph, as well as of a tree.
Regardless of their difference, all graph classes have methods for returning a list of the nodes; returning a list of the links; returning a list of nodes, connected to a given node; returning the dictionary of neighboring nodes for every node; returning the degree of a given node; returning the degrees of all nodes in the graph; adding a new node to the graph; removing a positive number of nodes from the graph; connecting a node to a positive number of other nodes in the graph; disconnecting a node from a positive number of other nodes in the graph; returning the complementary graph of the original; returning a copy of the graph; listing out the connection components in the graph; checking whether the graph is connected; checking whether two nodes are reachable in the graph; evaluating whether a path with a given length exists between two nodes; evaluating whether a loop with a given length exists in the graph at all; listing out bridge nodes in the graph; listing out bridge links in the graph; calculating the shortest path between two nodes in the graph; calculating only the length of the shortest path between two nodes in the graph; checking whether an Euler tour exists, whether an Euler walk exists, whether a Hamilton tour exists and whether a Hamilton walk exists; actually finding an Euler tour, an Euler walk, a Hamilton tour and a Hamilton walk; checking whether a node or a link is present in the graph; comparing whether two graphs are the same; combining two graphs into one (addition) and representing the graph.
Undirected graph classes further have methods for checking whether the graph could be a tree; listing out the cliques with a given number of nodes in the graph; finding the chromatic number of the nodes of the graph; finding the chromatic number of the links of the graph; evaluating the smallest number of holes on the surface, on which the graph can be drawn without any of its links intersecting; evaluating whether the graph is planar; counting how many faces the graph separates the surface with least holes, on which it can be drawn without any of its links intersecting, into and checking whether the graph is full (whether there are links between every two nodes in it), while the only method, unique for the directed graphs, is the one, that evaluates whether the graph is strongly connected (whether a path exists between exery two nodes in it).
However, there are differences in some of the common methods, individual to the directed graphs only, and they are, that the method for adding a new node accepts an iter, containing nodes, that point to the new node and an iter, containing nodes, that the new node points to; instead of one method for connecting nodes, there are two methods - connect_from_to and connect_to_from, that work as their names suggest; the degrees method returns a pair of numbers, the first of which shows how many nodes point to it and the second one shows how many nodes it points to, if a node is given, otherwise it returns the same for all nodes; the neighboring method only returns the nodes, pointed by a node, if such is given, otherwise it returns the same for all nodes and the method, that checks whether two nodes are reachable, uses BFS to check whether there is a path between the two given nodes.
Weighted graphs, in addition to their parental superclasses, have methods for returning the weight of a link, if such is given, otherwise returns the same for all links; returning the sum of the weights of all links; finding the minimal spanning tree of the graph and finding the lightest (or minimal) path between two nodes.
On top of that, the methods for adding and connecting nodes differ such, that instead of accepting a positive number of nodes, which a given node is going to be connected to,
they accept a positive number of pairs, each of which contains a node and a real number, that is going to be the value of the link between the two nodes.
Also, their methods for finding an Euler tour, an Euler walk, a Hamilton tour and a Hamilton walk also return the sum of the weights of the links in the paths found.
And finally, weighted graphs are represented with their nodes and their dictionary of the weights, rather than only the nodes and the links.
